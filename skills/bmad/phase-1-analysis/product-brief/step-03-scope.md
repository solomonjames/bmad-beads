## Instructions

### Part 1: MVP Core Features
1. Define essential features collaboratively:
   - What is the minimum needed to solve the core problem?
   - What features make users say "this solves my problem"?
   - Balance ambition with speed to value
2. Apply MVP criteria: Solves Core Problem, User Value, Feasible, Testable

### Part 2: Scope Boundaries
1. Define what is explicitly OUT of scope for MVP:
   - Nice-to-have features for later
   - Version 2.0 candidates
   - Intentional "nos" with rationale
2. Include clear communication of boundaries and trade-off explanations

### Part 3: MVP Success Criteria
1. Define what makes MVP successful:
   - User adoption metrics
   - Problem validation signals
   - Technical feasibility confirmation
   - Business model validation gates

### Part 4: Future Vision
1. Explore growth trajectory:
   - What does it become if wildly successful?
   - Post-MVP enhancements
   - Scale considerations
   - Ecosystem expansion opportunities
